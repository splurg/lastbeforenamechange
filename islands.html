Island / Partial SPA Architecture

Keep your HTML-first pages intact and progressively enhance only the high-interactivity regions by mounting tiny Preact components — called islands — directly onto server-rendered DOM nodes. The surrounding page never becomes a single-page app; it stays fast, accessible, and htmx-friendly.

How it works

Server-rendered HTML

Header, nav, hero, static content, footer — delivered as plain HTML; zero JS cost.

⚡ Preact Island

High-interactivity region (search, data table, counter…) hydrated client-side from a data-island attribute.

Server-rendered HTML

More static content — forms driven by htmx, accordions powered by hyperscript.

⚡ Preact Island

Another interactive region, independently bundled and lazy-loaded.

Key principles

  • Progressive enhancement — islands contain a static HTML fallback inside the mount node; Preact replaces it once JS loads.
  • Minimal footprint — Preact weighs ~3 KB (gzip). Each island is code-split and lazy-loaded only when needed.
  • Coexistence — htmx and hyperscript keep working on everything outside an island. Mix and match freely.
  • Accessibility — islands use semantic HTML, ARIA attributes, and keyboard-accessible controls; no SPA routing quirks.

Mount syntax

Add data-island and data-props attributes to any element:

<div
  data-island="LiveSearchIsland"
  data-props='{"items":[...],"heading":"Search programs"}'
>
  <!-- optional static fallback shown before JS loads -->
  <p>Loading search…</p>
</div>

The island loader (src/islands/mount.ts) scans for [data-island] nodes at page load, dynamically imports the matching component, and calls preact/render into the element.
